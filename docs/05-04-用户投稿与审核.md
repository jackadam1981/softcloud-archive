# 05-04 用户投稿与审核

> 阶段 4：用户投稿软件，管理员审核后发布。

---

## 目标与范围

- 用户投稿：提交软件信息，写入 software_submissions
- 用户查看：我的投稿列表及状态
- 管理员审核：待审核列表、通过/驳回
- 审核通过时可创建/关联 software 记录

---

## 依赖

- 阶段 1、2、3 已完成
- `software_submissions` 表已存在

---

## 任务清单

### Workers API

- [ ] `POST /api/software/submit` — 用户投稿，需 JWT，body: title, payload
- [ ] `GET /api/user/submissions` — 我的投稿列表，需 JWT
- [ ] `GET /api/admin/submissions` — 管理员待审核列表，query: status
- [ ] `POST /api/admin/submissions/:id/review` — 审核，body: status (approved|rejected), review_note

### 前端

- [ ] 「提交软件」页面：表单 + 登录校验
- [ ] 「我的投稿」页面：列表、状态、驳回原因
- [ ] 后台「用户投稿审核」：列表、通过/驳回操作
- [ ] 所有 admin 请求 `credentials: "include"`

---

| 文件 | 说明 |

---

## 验收标准

- 登录用户可提交投稿
- 用户可查看自己的投稿及审核状态
- 管理员可审核，通过/驳回后状态正确更新
