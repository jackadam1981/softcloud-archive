# 05-03 软件管理闭环

> 阶段 3：后台软件增删改查、编辑、软删除完整闭环。

---

## 目标与范围

- 后台软件列表、搜索、分页
- 新增软件（POST）
- 编辑软件（PUT），含 long_desc、版本、上下架
- 软删除（DELETE），设置 is_deleted=1
- 下载缺失链接时前端展示「暂不可下载」

---

## 依赖

- 阶段 1、2 已完成
- 管理认证：JWT（is_admin=1）或 Bearer ADMIN_TOKEN
- 接口形态与错误码见 [04-API合约](./04-API合约.md) §3

---

## 任务清单

### Workers API

- `GET /api/admin/software` — 列表，支持 q、page、includeDeleted
- `POST /api/admin/software` — 创建，必填 name、slug
- `PUT /api/admin/software/:id` — 更新，支持 is_published、is_featured 等
- `DELETE /api/admin/software/:id` — 软删除
- 下载接口返回 `DOWNLOAD_URL_MISSING` 时前端可识别

### 前端后台

- `/admin/software` 列表页，带搜索、新增按钮
- `/admin/software/new` 新增表单
- `/admin/software/:id` 详情页，支持编辑模式
- 编辑表单：名称、slug、版本、short_desc、long_desc、icon_url（见 [03 图片存储约定](./03-架构与开发顺序.md#6-图片存储约定)）、openlist_share_url、openlist_file_path、is_published
- 删除按钮（软删），需二次确认
- 请求时携带 `credentials: "include"` 以发送 Cookie

## 验收标准

- 管理员可增删改查软件
- 软删除后前台列表不展示，后台可查（includeDeleted=1）
- 下载链接缺失时，详情页显示「暂不可下载」
