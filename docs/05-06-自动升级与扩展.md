# 05-06 自动升级与扩展

> 阶段 6：软件更新源、定时任务、批量导入、统计 API。

---

## 目标与范围

- 软件更新源（software_sources）：GitHub、HTML 等类型配置
- 定时任务（Scheduled Worker）：批量检查更新源，检测新版本
- 批量导入：脚本扫描 OpenList 目录 + 元数据，生成 SQL 或调用 API 入库
- 统计 API：热门下载、全局统计

---

## 依赖

- 阶段 1～5 已完成

---

## 任务清单

### 数据库

- [ ] `software_sources` 表：software_id, source_type, config_json, enabled, last_checked_at
- [ ] `software_source_logs` 表：检测日志、结果、错误信息

### Workers

- [ ] `GET /api/admin/software/:id/sources` — 更新源列表
- [ ] `POST /api/admin/software/:id/sources` — 添加更新源
- [ ] `PUT /api/admin/software/:id/sources/:sourceId` — 更新
- [ ] `GET /api/admin/software/:id/source-logs` — 检测日志
- [ ] `scheduled`  handler：定期读取 software_sources，执行 GitHub/HTML 检测逻辑
- [ ] `GET /api/stats/top-downloads` — 热门下载（已有）
- [ ] （可选）`GET /api/stats/summary` — 全局统计

### 批量导入（简化版）

- [ ] 脚本或内部工具：扫描 OpenList 目录，匹配安装包 + `.txt` 元数据
- [ ] 输出 SQL 或 JSON，供人工确认后执行
- [ ] 不要求 import_jobs 表、导入 API、前端 UI（可后续升级）

### 前端

- [ ] 软件详情页展示更新源配置与检测日志
- [ ] 首页或列表页展示热门下载
