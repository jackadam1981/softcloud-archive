# 05-02 数据库与核心 API

> 阶段 2：建立核心数据模型与基础接口。

---

## 目标与范围

- 实现 `users`、`software`、`downloads` 等核心表的读写
- 认证：注册、登录、登出、JWT + HttpOnly Cookie
- 软件列表、详情、下载（302 至 OpenList）
- 前台只展示 `is_published=1 AND is_deleted=0` 的软件

---

## 依赖

- 阶段 1（环境与基础设施）已完成
- 实施前建议阅读 [04-API合约](./04-API合约.md)，作为接口形态与错误码参考

---

## 任务清单

### 数据库

- [ ] `users` 表：id, email, password_hash, nickname, is_admin, created_at
- [ ] `software` 表：name, slug, version, short_desc, long_desc, openlist_share_url, openlist_file_path, is_published, is_deleted 等
- [ ] `downloads` 表：software_id, user_id, ip, ua, created_at
- [ ] 必要索引：email、slug、is_published、is_deleted

### Workers API

- [ ] `POST /api/auth/register` — 注册，第一个用户自动 is_admin=1
- [ ] `POST /api/auth/login` — 登录，JWT 写入 HttpOnly Cookie
- [ ] `POST /api/auth/logout` — 清除 Cookie
- [ ] `GET /api/auth/profile`、`PUT /api/auth/profile` — 获取/更新个人资料
- [ ] `GET /api/software` — 列表，支持 q、category、platform、分页
- [ ] `GET /api/software/:slug` — 详情（仅已发布且未删除）
- [ ] `POST /api/software/:slug/download` — 记录下载 + 302 至 openlist_share_url，缺失时返回 `DOWNLOAD_URL_MISSING`

### 前端

- [ ] 首页、软件列表、软件详情、登录/注册页
- [ ] 下载按钮调用 download 接口，处理 302 与错误码

## 验收标准

- 可注册、登录，Cookie 正确设置
- 软件列表与详情可正常访问
- 有 share_url 时点击下载可 302 到 OpenList；无时返回 400 并提示
