# SoftCloud Archive 软件需求设计分析书

> 需求分析 + 高层设计，回答「要满足什么需求、如何设计」。

---

## 1. 功能需求

### 1.1 访客

| 需求 | 描述 |
|------|------|
| 浏览软件 | 按分类、平台、关键字查看软件列表，支持分页 |
| 搜索软件 | 按名称、简介搜索 |
| 查看详情 | 软件名称、版本、简介、长描述（Markdown）、下载按钮 |
| 下载 | 点击下载 → 记录日志 → 302 跳转至网盘分享链接（用户直接访问网盘） |

### 1.2 注册用户

| 需求 | 描述 |
|------|------|
| 注册 | 邮箱 + 密码，密码需包含字母和数字、至少 8 位 |
| 登录 | 邮箱 + 密码，登录后 JWT 写入 HttpOnly Cookie |
| 修改资料 | 昵称等可修改字段 |
| 投稿软件 | 提交软件信息，状态为待审核，管理员审核后对外展示 |

### 1.3 管理员

| 需求 | 描述 |
|------|------|
| 软件 CRUD | 新增、编辑、删除（软删除）、上下架 |
| 存储配置 | 管理存储后端、软件-文件关联、多后端容量视图 |
| 审核投稿 | 查看待审核列表，通过/驳回 |
| 更新源配置 | 配置 GitHub/HTML 等更新源，定时检测新版本（规划） |

## 2. 非功能需求

### 2.1 性能

- 前端、API 部署在 Cloudflare 边缘
- D1 为边缘数据库，低延迟
- 大文件不经过 Workers，用户通过分享链接直接从网盘下载

### 2.2 安全

- 用户：JWT + HttpOnly Cookie，密钥由 `JWT_SECRET` 管理
- 管理接口：JWT（is_admin=1）或 Bearer `ADMIN_TOKEN` / `admin_tokens` 表
- 软删除：不物理删除，避免数据孤儿
- 下载限流：按 IP 限制请求频率，防止滥用

### 2.3 可维护性

- API 合约文档化，错误码统一
- 多语言 i18n，所有文案从语言文件加载
- 日志：登录失败、后台操作、下载异常等关键路径有结构化日志

## 3. 数据流概览

### 3.1 下载流程

```
用户点击下载 → 前端请求 POST /api/software/:slug/download
→ Workers 校验软件存在且已发布
→ 写入 downloads 表
→ 取 openlist_share_url（或 software_files 主文件）
→ 有链接：302 重定向至网盘分享链接（用户直接从网盘下载）
→ 无链接：400 { "error": "DOWNLOAD_URL_MISSING" }
```

### 3.2 管理流程

```
后台页面 → 请求 /api/admin/*（带 JWT Cookie 或 Authorization: Bearer）
→ requireAdmin 校验（JWT is_admin=1 或 Bearer token）
→ Workers 读写 D1
→ 返回 JSON
```

### 3.3 投稿与审核流程

```
用户投稿 → POST /api/software/submit（需 JWT）
→ 写入 software_submissions
→ 管理员 GET /api/admin/submissions?status=pending
→ POST /api/admin/submissions/:id/review { status: "approved" | "rejected" }
→ 审核通过时可创建/更新 software 记录
```

## 4. 边界与依赖

### 4.1 职责边界

| 组件 | 职责 | 不负责 |
|------|------|--------|
| 前端 (Nuxt) | 展示、路由、i18n、主题 | 不直连 D1/OpenList |
| Workers | 业务逻辑、鉴权、下载跳转、API | 不存储大文件 |
| D1 | 结构化数据（用户、软件、下载日志等） | - |
| OpenList | 网盘聚合、WebDAV 上传、分享链接生成 | - |

### 4.2 关键约定

- 前端只通过 Workers API 获取/提交数据
- Workers 只存 OpenList 的 `share_url`、`file_path`，不存文件内容
- 前台列表/详情一律过滤 `is_published=1 AND is_deleted=0`
- 错误响应统一为 `{ "error": "ERROR_CODE" }`，详见 [04-API合约](./04-API合约.md)
